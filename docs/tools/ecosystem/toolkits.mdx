---
title: "Toolkits"
description: "Pre-built tool collections for Beige agents"
---

Toolkits are multi-tool repositories published by the community. They bundle several related tools together for easy installation.

## Installing a Toolkit

```bash
# From npm
beige tools install npm:@matthias-hausberger/beige-toolkit

# From GitHub
beige tools install github:matthias-hausberger/beige-toolkit

# Specific version
beige tools install npm:@matthias-hausberger/beige-toolkit@0.1.2
```

All tools in the toolkit are installed to `~/.beige/tools/` and auto-discovered by the gateway.

## Using Toolkit Tools

After installing, add tools to your agent's `tools` array:

```json5
{
  agents: {
    assistant: {
      tools: ["github", "chrome", "slack", "sessions"],
    },
  },
}
```

No `path` or `target` needed — installed tools are resolved automatically. Add a `tools.<name>` entry only for custom config:

```json5
{
  tools: {
    github: {
      config: { allowedCommands: ["repo", "issue", "pr"] },
    },
    slack: {
      config: { denyCommands: ["messages send", "messages draft"] },
    },
  },
}
```

## Cherry-Picking Individual Tools

If you only need specific tools from a toolkit, install them individually via GitHub:

```bash
beige tools install github:matthias-hausberger/beige-toolkit/tools/github
beige tools install github:matthias-hausberger/beige-toolkit/tools/chrome
```

## Managing Toolkit Tools

Toolkit tools are managed the same way as any installed tool:

```bash
beige tools list                    # See all installed tools
beige tools update github           # Update (re-fetches from original source)
beige tools update                  # Update all tools
beige tools remove github           # Remove a specific tool
```

See [Managing Tools](/tools/managing) for details.
