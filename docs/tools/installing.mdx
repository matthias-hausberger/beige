---
title: "Installing Tools"
description: "Install tools from npm, GitHub, or local directories"
---

Tools are installed with `beige tools install`. Each installed tool is placed in `~/.beige/tools/<name>/` and automatically discovered by the gateway at startup.

## Install Sources

### From npm

Install a published npm package containing one or more tools:

```bash
# Latest version
beige tools install npm:@matthias-hausberger/beige-toolkit

# Specific version
beige tools install npm:@matthias-hausberger/beige-toolkit@0.1.2

# Pre-release version
beige tools install npm:@matthias-hausberger/beige-toolkit@0.2.0-beta.3

# Unscoped package
beige tools install npm:my-tool-package
```

npm packages can contain a single tool (with `tool.json` at the package root) or multiple tools (with `tool.json` files in subdirectories). All tools in the package are installed.

### From GitHub

Install all tools from a repository:

```bash
# Default branch
beige tools install github:owner/repo

# Specific tag or branch
beige tools install github:owner/repo#v1.0.0
beige tools install github:owner/repo#main
```

Install a single tool from a subfolder:

```bash
beige tools install github:owner/beige-toolkit/tools/github
beige tools install github:owner/beige-toolkit/tools/chrome
```

The subfolder must contain a `tool.json` file. This lets you cherry-pick individual tools from a multi-tool repository.

### From a Local Directory

```bash
# Relative path
beige tools install ./my-tool

# Absolute path
beige tools install /home/user/projects/my-tool
```

For single-tool directories (with `tool.json` at root), the tool is copied to `~/.beige/tools/`. For directories containing multiple tools, the directory is stored in `~/.beige/packages/` and each tool is symlinked.

## How It Works

### Single tool

When the install source contains exactly one `tool.json` at its root:

1. The tool directory is copied to `~/.beige/tools/<name>/`
2. If a `package.json` with `dependencies` exists, `npm install --production` is run
3. A `<name>.meta.json` file is written alongside with the install source for updates

```
~/.beige/tools/
  my-tool/
    tool.json
    index.ts
    package.json        # optional — only if the tool has npm dependencies
    node_modules/       # created by npm install if needed
  my-tool.meta.json     # { "source": "github:owner/repo/tools/my-tool", ... }
```

### Multi-tool package

When the install source contains multiple `tool.json` files (e.g., an npm package or a GitHub repo with several tools):

1. The entire package is stored in `~/.beige/packages/<name>/`
2. Dependencies are installed per-tool (each tool's own `package.json`)
3. Each tool is symlinked into `~/.beige/tools/`
4. Meta files are written for each tool

```
~/.beige/
  packages/
    matthias-hausberger-beige-toolkit/
      tools/
        github/
          tool.json
          index.ts
        chrome/
          tool.json
          index.ts
          mcp-client.ts
          node_modules/    # chrome has npm dependencies
  tools/
    github → ../packages/matthias-hausberger-beige-toolkit/tools/github
    github.meta.json
    chrome → ../packages/matthias-hausberger-beige-toolkit/tools/chrome
    chrome.meta.json
```

Symlinks ensure that tools can resolve their npm dependencies correctly — Node.js resolves modules from the real path, so `node_modules` in the package directory is found.

## Handling Conflicts

If a tool with the same name is already installed, the install is rejected:

```
[BEIGE] Tool name conflicts detected:
  - Tool 'github' already installed (from github:owner/other-repo)

[BEIGE] Use --force to override.
```

Use `--force` to replace conflicting tools:

```bash
beige tools install github:owner/new-repo --force
```

## Dependencies

Tools can have npm dependencies declared in their own `package.json`:

```json
{
  "name": "@beige/tool-chrome",
  "dependencies": {
    "some-mcp-lib": "^1.0.0"
  }
}
```

The installer runs `npm install --production` per tool directory. This works regardless of what package manager the tool author uses for development (pnpm, yarn, npm) — the `package.json` just needs standard `dependencies`.

<Note>
Pin exact versions in `dependencies` (e.g., `"1.2.3"` instead of `"^1.2.3"`) for maximum reproducibility, since installed tools don't include lockfiles.
</Note>

## After Installing

After installation, add tools to your agent's `tools` array in `config.json5`:

```json5
{
  agents: {
    assistant: {
      tools: ["github", "chrome", "slack"],
    },
  },
}
```

Then restart the gateway:

```bash
beige gateway restart
```

Installed tools are auto-discovered — you don't need to specify `path` or `target` in the config. Only add a `tools.<name>` entry if you need to pass custom config:

```json5
{
  tools: {
    github: {
      config: { allowedCommands: ["repo", "issue", "pr"] },
    },
  },
}
```
