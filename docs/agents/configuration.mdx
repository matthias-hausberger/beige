Beige is configured with a single `~/.beige/config.json5` file. JSON5 is JSON with comments, trailing commas, and unquoted keys. You can also override the path with `--config`.

## Editor Autocomplete

Add a `$schema` line to get inline autocomplete and validation in VS Code, JetBrains, and any editor that supports JSON Schema:

```json5
{
  $schema: "https://raw.githubusercontent.com/matthias-hausberger/beige/main/config.schema.json",
  // ... rest of config
}
```

The schema is generated from the TypeBox definitions in `src/config/schema.ts` and committed at `config.schema.json` in the repository. `beige setup` writes this line automatically into the generated config.

## Environment Variables

Any string value can reference environment variables using `${VAR_NAME}`. Variables are resolved at startup вЂ” if a referenced variable is not set, the gateway exits with an error.

```json5
apiKey: "${ANTHROPIC_API_KEY}",  // resolved from env at startup
```

## Tool Path Resolution

Tool `path` values are resolved relative to the config file's directory, unless they are absolute paths.

| Install mode | Recommended path style |
|---|---|
| **npm global** | Absolute (`/Users/you/.beige/tools/kv`) вЂ” written by `beige setup` |
| **Source / git clone** | Absolute or repo-relative (`./tools/kv`) |

## Full Config Reference

```json5
{
  // в”Ђв”Ђ LLM Providers в”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђ
  // Configure one or more LLM providers. Each key becomes a provider name
  // that agents reference in their model config.
  llm: {
    providers: {
      // Built-in providers only need an apiKey.
      anthropic: {
        apiKey: "${ANTHROPIC_API_KEY}",
      },
      openai: {
        apiKey: "${OPENAI_API_KEY}",
      },
      google: {
        apiKey: "${GOOGLE_API_KEY}",
      },
      // Custom / OpenAI-compatible providers also need baseUrl and api.
      zai: {
        apiKey: "${ZAI_API_KEY}",
        baseUrl: "https://api.zai.com/v1",
        api: "openai-completions",
      },
      // Local (no apiKey needed)
      ollama: {
        baseUrl: "http://localhost:11434/v1",
        api: "openai-completions",
      },
    },
  },

  // в”Ђв”Ђ Tool Registry в”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђ
  // Register tool packages. Each key becomes the tool name used in agent
  // configs, launcher scripts (/tools/bin/<name>), and audit logs.
  tools: {
    kv: {
      path: "~/.beige/tools/kv",  // path to tool package directory
      target: "gateway",           // where the handler runs: "gateway" | "sandbox"(future)
      config: {},                  // arbitrary config passed to createHandler()
    },
  },

  // в”Ђв”Ђ Skill Registry в”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђ
  // Register skill packages (read-only knowledge mounted at /skills/<name>/).
  skills: {
    "code-review": {
      path: "./skills/code-review",  // path to skill package directory
    },
  },

  // в”Ђв”Ђ Agents в”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђ
  // Each agent gets its own Docker sandbox, Unix socket, and LLM session.
  agents: {
    assistant: {
      model: {
        provider: "anthropic",                    // must match a key in llm.providers
        model: "claude-sonnet-4-6",        // model ID as the provider expects it
        thinkingLevel: "off",                     // off | minimal | low | medium | high
      },
      fallbackModels: [                           // tried in order if primary fails
        { provider: "anthropic", model: "claude-3-5-sonnet-20241022" },
      ],
      tools: ["kv"],                              // tool names from the tools registry above
      skills: ["code-review"],                    // skill names from the skills registry above
      sandbox: {
        image: "beige-sandbox:latest",            // Docker image (default)
        extraMounts: {                            // additional hostв†’container bind mounts
          "/host/path": "/container/path",        // reduces isolation вЂ” use carefully
        },
        extraEnv: {                               // env vars for the container
          "NODE_ENV": "production",               // never use for secrets вЂ” agent can read these
        },
      },
    },
  },

  // в”Ђв”Ђ Gateway в”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђ
  // All fields are optional; defaults shown.
  gateway: {
    host: "127.0.0.1",                           // HTTP API bind address
    port: 7433,                                   // HTTP API port
    logFile: "~/.beige/logs/gateway.log",         // daemon log file
  },

  // в”Ђв”Ђ Channels в”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђ
  channels: {
    telegram: {
      enabled: true,
      token: "${TELEGRAM_BOT_TOKEN}",
      allowedUsers: [123456789],                  // Telegram user IDs allowed to interact
      agentMapping: {
        default: "assistant",                     // agent that handles messages
      },
      defaults: {
        verbose: false,                           // send tool-call notifications by default
      },
    },
  },
}
```

## Field Reference

### `llm.providers.<name>`

| Field | Required | Description |
|-------|----------|-------------|
| `apiKey` | Yes* | API key вЂ” use `${VAR}` for env var. Not required for local providers. |
| `baseUrl` | No | Custom API endpoint. Required for non-built-in providers. |
| `api` | No | API type (see below). Auto-set for built-in providers. |

**API types:**

| Value | Use for |
|-------|---------|
| `anthropic-messages` | Anthropic Claude (default for `anthropic`) |
| `openai-completions` | OpenAI, ZAI, Groq, Ollama, and most OpenAI-compatible APIs |
| `openai-responses` | OpenAI Responses API |
| `google-generative-ai` | Google Gemini (default for `google`) |

### `tools.<name>`

| Field | Required | Description |
|-------|----------|-------------|
| `path` | Yes | Path to the tool package directory |
| `target` | Yes | `"gateway"` (runs on host) вЂ” `"sandbox"` is future |
| `config` | No | Arbitrary object passed to `createHandler(config)` |

### `skills.<name>`

| Field | Required | Description |
|-------|----------|-------------|
| `path` | Yes | Path to the skill package directory |

### `agents.<name>.model`

| Field | Required | Description |
|-------|----------|-------------|
| `provider` | Yes | Must match a key in `llm.providers` |
| `model` | Yes | Model ID as the provider expects it |
| `thinkingLevel` | No | `off` \| `minimal` \| `low` \| `medium` \| `high` вЂ” extended thinking budget |

### `agents.<name>.sandbox`

| Field | Required | Description |
|-------|----------|-------------|
| `image` | No | Docker image name (default: `beige-sandbox:latest`) |
| `extraMounts` | No | `{ "/host/path": "/container/path" }` вЂ” reduces isolation |
| `extraEnv` | No | `{ "KEY": "value" }` вЂ” visible to the agent, never use for secrets |

### `gateway`

| Field | Default | Description |
|-------|---------|-------------|
| `host` | `"127.0.0.1"` | HTTP API bind address |
| `port` | `7433` | HTTP API port |
| `logFile` | `~/.beige/logs/gateway.log` | Daemon stdout/stderr log file |

### `channels.telegram`

| Field | Required | Description |
|-------|----------|-------------|
| `enabled` | Yes | `true` to activate the Telegram bot |
| `token` | Yes | Telegram bot token from BotFather |
| `allowedUsers` | Yes | Array of Telegram user IDs permitted to interact |
| `agentMapping.default` | Yes | Agent name to handle all incoming messages |
| `defaults.verbose` | No | Default `false` вЂ” when `true`, bot sends tool-call notifications |

## Validation at Startup

The gateway checks the following at startup and exits with an error if any check fails:

| Check | Error if |
|-------|----------|
| `llm.providers` exists | Missing or empty |
| `agents` exists | Missing or empty |
| Each agent has `model.provider` + `model.model` | Missing model config |
| Agent's `model.provider` exists in `llm.providers` | Unknown provider |
| Agent's tools all exist in `tools` registry | Unknown tool |
| Agent's skills all exist in `skills` registry | Unknown skill |
| Skill's `requires.tools` all in agent's `tools` | Missing required tool |
| Skill's `requires.skills` all in agent's `skills` | Missing required skill |
| Telegram `agentMapping.default` exists | Points to unknown agent |
| All `${VAR}` references resolve | Environment variable not set |

## Minimal Working Config

The smallest config that will start:

```json5
{
  llm: {
    providers: {
      anthropic: { apiKey: "${ANTHROPIC_API_KEY}" },
    },
  },
  tools: {},
  agents: {
    bot: {
      model: { provider: "anthropic", model: "claude-sonnet-4-6" },
      tools: [],
    },
  },
}
```

This agent has no custom tools but can still use the 4 core tools (`read`, `write`, `patch`, `exec`) inside its sandbox.
